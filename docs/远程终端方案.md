# 手机远程控制 Mac 终端（kimi / claude / tmux）方案文档

一套自建的"安卓手机 → 远程操控 macOS 上的 kimi CLI、claude code、tmux 会话"方案。
全链路：安卓 App（原生终端仿真）→ Tailscale 加密组网 → Mac 上的 WebSocket-PTY 服务端 → kimi / claude / tmux。

## 1. 架构总览

```
安卓手机                          家里电脑 (macOS 15, arm64)
┌─────────────────┐   Tailscale   ┌──────────────────────────────────┐
│ 远程终端 App     │  加密隧道      │ remote-terminal server (Node.js)  │
│ (Kotlin + Termux│ ────────────▶ │ 0.0.0.0:7681, token 鉴权          │
│  terminal-view) │  ws://100.x.x.x│   ├─ app=kimi   → 新 kimi 进程     │
│ 多标签 / 触控    │   :7681/ws    │   ├─ app=claude → 新 claude 进程   │
└─────────────────┘               │   └─ app=tmux   → tmux new-session │
                                  │                       -A -s <会话> │
                                  │ 每个连接 = 一个独立 PTY (node-pty) │
                                  └──────────────────────────────────┘
                                           ▲ launchd 开机自启 + 崩溃重拉
```

- 每个手机标签页 ↔ 服务端一条 WebSocket ↔ Mac 上一个独立 PTY 进程
- 协议：连接 `ws://<host>:7681/ws?app=<kimi|claude|tmux>&token=<token>&cols=N&rows=N[&session=名字]`；
  二进制帧双向传终端字节流；文本帧 JSON 传 `resize` / `exit` / `error`；
  健康检查 `GET /health?token=...`
- tmux 接入语义：`tmux new-session -A -s <名字>` —— 会话已存在则接入，不存在则新建持久会话

## 2. 组件清单（全部在 ~/remote-terminal/ 内，自包含）

| 组件 | 路径 | 说明 |
|---|---|---|
| 服务端 | `~/remote-terminal/server/server.js` | Node 24 + node-pty + ws，源码见注释 |
| 服务端配置 | `~/remote-terminal/server/config.json` | 端口、**token**、app 白名单（含 tmux 路径）。mode 600 |
| 自启动 | `~/Library/LaunchAgents/com.calla.remote-terminal.plist` | RunAtLoad + KeepAlive，日志在 `server/launchd.{out,err}.log` |
| 安卓 App 源码 | `~/remote-terminal/android-app/` | Kotlin，Termux terminal-view v0.118.1（JitPack）+ OkHttp |
| 构建环境 | `~/remote-terminal/{jdk,android-sdk,gradle}` | JDK 17 / SDK 34 / Gradle 8.9，`source env.sh` 即可用 |
| 构建脚本 | `~/remote-terminal/env.sh` | JAVA_HOME / ANDROID_HOME / PATH |
| APK 产物 | `~/Downloads/app-debug.apk` | 覆盖安装到手机即可 |

相关本机改动：`~/.tmux.conf` 末尾追加了"拖选复制进 macOS 剪贴板"的绑定
（`copy-pipe-and-cancel pbcopy`，解决 tmux 默认 OSC52 在部分终端不同步剪贴板的问题）。

## 3. 网络：Tailscale 组网

- 国区 App Store 无 Tailscale；Mac 用官网独立安装包（`~/Downloads/Tailscale-*-macos.pkg` 或
  https://pkgs.tailscale.com/stable/#macos），安卓用 Google Play 或
  https://github.com/tailscale/tailscale-android/releases 的 APK
- 两端登录**同一账号**，手机 App 里"主机"填 Mac 的 `100.x.x.x` 地址（菜单栏 Tailscale 图标可见）
- 在家时也可以直接填局域网 IP（如 192.168.1.10），不经 Tailscale
- 安卓上 Tailscale 占用系统 VPN 通道，与 VPN 类代理 App 互斥；登录授权页（尤其 Google）打不开时，
  临时开代理登录，成功后关掉切回 Tailscale

## 4. 手机端使用

1. 安装 `app-debug.apk`（允许"安装未知来源应用"）
2. 启动页填：主机 `100.x.x.x` / 端口 `7681` / Token（见 `server/config.json`，可自取可改）→
   先点"测试连接"，成功后选程序连接
3. 程序选择：
   - **kimi / claude**：各开一个全新进程（多开互不干扰）
   - **tmux**：填会话名。填 `kimi1`/`kimi2` 接入 Mac 上已有会话（含 5h/7d 用量状态栏）；
     默认 `phone` 为手机专用持久会话，断开后仍存活，重连继续
4. 多标签：顶部标签栏，「+」新建（kimi/claude/tmux 任选），点按切换，长按关闭；
   后台标签连接保持存活，断连标签置灰标 ✕
5. 触控：
   - 点终端画面下半部（输入区）自动弹软键盘
   - 方向键/ESC/CTRL/TAB 在底部特殊键栏；CTRL 为粘滞键（按一下生效一次，如 CTRL→C = 0x03）
   - 终端鼠标上报（tmux 点 pane、TUI 点选项）走 xterm SGR 1006 标准；
     **被控程序是否响应点击取决于它自身**（tmux 已开 mouse on，可点窗格/状态栏/滚屏）
   - 「横屏」按钮切换横竖屏；双指缩放字号（按标签各自记忆）
6. tmux 内进阶：prefix+(/) 切换会话，prefix+d 脱离（会话保留），exit 结束会话

## 5. Mac 端运维

```bash
# 重启 / 停止 / 状态
launchctl kickstart -k gui/$(id -u)/com.calla.remote-terminal
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.calla.remote-terminal.plist   # 停用并卸载
launchctl print gui/$(id -u)/com.calla.remote-terminal | head

# 日志
tail -f ~/remote-terminal/server/launchd.out.log     # 连接/退出记录
tail -f ~/remote-terminal/server/launchd.err.log

# 改 token：编辑 ~/remote-terminal/server/config.json 后重启服务（上面第一条）
# 手动前台调试运行：
pkill -f "remote-terminal/server/server.js"; cd ~/remote-terminal/server && node server.js
```

**Mac 不能睡眠**，否则外出连不上：系统设置 → 电池/电源 → 接通电源时防止自动睡眠。

## 6. 重新构建 APK

```bash
source ~/remote-terminal/env.sh
cd ~/remote-terminal/android-app
gradle assembleDebug        # 或 ./gradlew assembleDebug
cp app/build/outputs/apk/debug/app-debug.apk ~/Downloads/
```

首次构建会从 JitPack/Maven 拉依赖（terminal-view 的 JitPack 产物为 v0.118.1，含四 ABI 的 libtermux.so）。

## 7. 已知事项与注意

- **共享 attach 会缩窗**：手机 attach 到 Mac 正使用的 tmux 会话时，tmux 窗口尺寸取所有客户端最小值，
  Mac 本地窗口会临时缩成手机大小，手机断开后恢复。介意就用独立会话名
- **token 即全部鉴权**：`config.json` 含明文 token，分享本目录/压缩包前注意；泄露后改 token 并重启服务
- **服务端重启会断开所有手机连接**（各会话若是 tmux 则存活，kimi/claude 裸进程会被杀）
- EADDRINUSE 恢复：若重启后连不上，`lsof -nP -iTCP:7681 -sTCP:LISTEN` 找到旧 PID 杀掉，
  launchd 会自动拉起新进程
- App 为 debug 签名，无图标；分发需自行 release 签名
- 输入法兼容：App 内 `shouldEnforceCharBasedInput=true`，个别国产输入法异常时改为 false 重建
